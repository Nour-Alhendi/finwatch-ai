## Overview

This project impelements a modular monitoring system for financial time-series data. 
It is my final project at WBS Coding School.

The system detects anomalies, evaluates their severity and impact, and applies controlled decision logic to determine whether to fix, preserve, or excalate unusual events. 

The goal is to move from simple anomaly detection to a stuctured, decision-driven monitoring framework. 
 -----------
## Proplem Statment

Financial time-series data is critical for risk management, analytics, and decision-making.
Data errors, regime shifts, and extreme events can distort models and trigger incorrect actions. 

Traditional anomaly detection systems only flag unusual values but do not decide what to do next. 
 
This project introduces a structured monitoring and decision pipeline to adress this gap. 
 -----------

## System Architecture

The system follows a modular pipeline:
1. Data Ingestion
2. Data Quality Checks
3. Feature Engineering
4. Anomaly Detection
5. Severity & Impact Scoring
6. Context Building
7. Guardails 
8. AI Decision Layer
9. Action Layer
10. Logging & Audit
